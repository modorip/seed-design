# 기여 가이드

## 환영합니다

모두립 디자인 시스템에 기여를 고려해주셔서 감사합니다.

## 어떤 기여가 가능한가요

- **새 토큰 제안**: `tokens/` 안의 명세에 새 값 추가
- **새 컴포넌트 추가**: `primitives/` 또는 `patterns/`
- **버그 / 문서 오타 수정**: 누구나 환영
- **시각 자료 보강**: 컴포넌트별 스크린샷·예제

## 기여 절차

1. **이슈로 먼저 논의**: 변경 의도와 사용 예를 공유합니다. 동의된 변경만 PR로 진행해주세요.
2. **포크 후 브랜치**: `feat/`, `fix/`, `docs/` prefix를 권장합니다.
3. **변경 단위는 작게**: 1 PR = 1 의도.
4. **PR 본문에 ADR이 필요한 변경인지 표시**: 토큰 의미 변경, 컴포넌트 API 변경은 ADR 동반.

## 카피·톤 가이드

- em-dash (`—`) 사용 금지
- 이모지 사용 금지
- "혁신적", "최고의" 등 과장 형용사 금지
- 분류학 용어("계열")는 사용자 노출 카피에서 회피

## 코드 스타일 (Phase 1 패키지화 시점)

- TypeScript strict mode
- React 18+
- 함수형 컴포넌트, hooks
- forwardRef + as prop 패턴
- CSS Variables 토큰 우선 (인라인 색 금지)
